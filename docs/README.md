# Документация проекта

Эта папка содержит документацию проекта Telegram бота.

## Структура документации

- **bot/** — документация по боту:
  - **BOT_REQUIREMENTS.md** — требования к функционалу
  - **BOT_FLOW.md** — пошаговый сценарий диалога
  - **BOTFATHER_SETUP.md** — настройка в BotFather
- **product_descriptions/** — описания продукта:
  - **PRESENTATION_FULL_DESCRIPTION_INTERVIEWS.md** — описание с учётом интервью
  - **PRESENTATION_LANDING.md** — лендинг продукта
- **PRODUCT_DESCRIPTION.md** - Описание продукта, целевая аудитория, преимущества
- **FEATURES.md** - Список реализованных и планируемых функций
- **USER_GUIDE.md** - Руководство пользователя, инструкции по использованию
- **ARCHITECTURE.md** - Архитектура приложения, структура кода
- **API.md** - Документация API (если применимо)
- **GITHUB_SETUP.md** - Инструкция по подключению GitHub и настройке релизов

## Как использовать

1. Заполните шаблоны информацией о вашем проекте
2. Обновляйте документацию по мере развития проекта
3. Используйте документацию для онбординга новых разработчиков

## Обновление документации

Документация должна обновляться при:
- Добавлении новых функций
- Изменении архитектуры
- Обновлении API
- Изменении пользовательского интерфейса
